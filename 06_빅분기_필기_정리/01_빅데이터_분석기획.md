## 1. 빅데이터 개요

### 3V → 5V
| V | 의미 |
|---|------|
| Volume | 데이터 양 |
| Velocity | 생성·처리 속도 |
| Variety | 형태 다양성(정형·반정형·비정형) |
| Veracity | 신뢰성 |
| Value | 가치 |

### 데이터 유형
- **정형**: DB, CSV, 테이블
- **반정형**: JSON, XML
- **비정형**: 텍스트, 이미지, 음성

**예제**  
① [부정형] 빅데이터의 3V에 해당하지 않는 것은?
- ① Volume
- ② Velocity
- ③ Variety
- ④ Variance  
> **정답** ④  
> **해설** 3V는 Volume, Velocity, Variety. 5V에 Veracity, Value 추가

② [정형] JSON, XML 형태의 데이터는?
- ① 정형
- ② 반정형
- ③ 비정형
- ④ 구조형  
> **정답** ②  
> **해설** 반정형 데이터

②-1 [O/X] ㉠ 정형 데이터는 CSV, DB 등이다. ㉡ 비정형은 텍스트, 이미지이다. 옳은 것만?
- ① ㉠
- ② ㉡
- ③ ㉠㉡
- ④ 없음  
> **정답** ③  
> **해설** ㉠㉡ 모두 옳음

## 2. 빅데이터 플랫폼

### Hadoop 생태계
- **HDFS**: 분산 파일 시스템
- **MapReduce**: 분산 처리
- **Hive**: SQL 쿼리
- **Spark**: 인메모리 처리(빠름)

### NoSQL
- **키-값**: Redis
- **문서**: MongoDB
- **컬럼**: HBase
- **그래프**: Neo4j

**예제**  
③ [정형] HDFS에 대한 설명으로 옳은 것은?
- ① SQL 쿼리 지원
- ② 인메모리 처리
- ③ 분산 파일 시스템
- ④ 문서형 DB  
> **정답** ③  
> **해설** HDFS = Hadoop Distributed File System

④ [정형] MongoDB는 NoSQL 중 어떤 유형인가?
- ① 키-값
- ② 문서
- ③ 컬럼
- ④ 그래프  
> **정답** ②  
> **해설** MongoDB는 문서형 NoSQL

④-1 [짝짓기] NoSQL (가)Redis (나)MongoDB (다)Neo4j — 유형 연결로 옳은 것은?
- ① (가)키-값 (나)문서 (다)그래프
- ② (가)문서 (나)그래프 (다)키-값
- ③ (가)그래프 (나)키-값 (다)문서  
> **정답** ①  
> **해설** Redis=키-값, MongoDB=문서, Neo4j=그래프

## 3. 개인정보·법·제도

### 개인정보보호법
- **개인정보**: 성명, 주민번호, 전화번호 등 식별 가능 정보
- **가명정보**: 추가 정보 없이 특정 개인 식별 불가
- **익명정보**: 복원 불가능하게 처리

### 동의·목적
- 수집·이용 목적 명시
- 최소한의 수집
- 제3자 제공 시 별도 동의

**예제**  
⑤ 가명정보에 대한 설명으로 옳은 것은?
- ① 복원 불가능
- ② 추가 정보 없이 특정 개인 식별 불가
- ③ 성명·주민번호 포함
- ④ 제3자 제공 시 동의 불필요  
> **정답** ②  
> **해설** 가명정보 = 추가 정보 없이는 식별 불가. 익명정보 = 복원 불가

## 4. 분석방안 수립

### 분석 절차
1. **문제 정의** → 2. **데이터 수집** → 3. **전처리** → 4. **탐색·모델링** → 5. **평가·해석**

### 데이터 수집 계획
- 내부 데이터 / 외부 데이터
- API, 웹 스크래핑, 공개 데이터셋

**예제**  
⑥ 데이터 분석 절차의 올바른 순서는?
- ① 문제정의→전처리→수집→모델링→해석
- ② 문제정의→수집→전처리→탐색·모델링→평가·해석
- ③ 수집→전처리→문제정의→모델링
- ④ 전처리→수집→모델링→해석  
> **정답** ②  
> **해설** 문제정의 → 수집 → 전처리 → 탐색·모델링 → 평가·해석
