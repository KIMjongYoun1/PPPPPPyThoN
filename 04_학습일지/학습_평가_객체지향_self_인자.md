### Q3: 코딩에서 "인자"라는 개념에 대해 설명해줘. self는 public 같은 접근 제어 인자인가?
**질문 수준:** ⭐⭐⭐⭐ (개념 혼동 발견 및 정정 요청)  
**이해 수준:** ⭐⭐⭐⭐⭐ (혼동 해소, 명확히 구분)

**질문 내용:**
- 인자(argument/parameter) 개념
- `self`가 접근 제어자인지 오해

**답변 요약:**
- **인자(Argument/Parameter):**
  - Parameter: 함수 정의 시 선언된 변수
  - Argument: 함수 호출 시 전달하는 실제 값
- **`self`는 접근 제어자가 아님:**
  - `self`: 객체 인스턴스 참조 (Java의 `this`)
  - 접근 제어: `_` 갯수로 표현 (`_` 0개=public, 1개=protected, 2개=private)

**이해도 평가:**
- ✅ 인자 개념 완전 이해
- ✅ `self`와 접근 제어자 명확히 구분
- ✅ Java의 `this`와 비교하여 이해

- ✅ 에러 발생 원인 완전 이해

## 📚 학습 내용 정리

### 1. 인자(Argument/Parameter) 개념

```python
# Parameter (매개변수): 함수 정의 시 선언
def greet(name):  # name = 매개변수
    print(f"안녕하세요, {name}!")

# Argument (인자): 함수 호출 시 전달하는 실제 값
greet("김철수")  # "김철수" = 인자
```

**핵심 이해:**
- Parameter = 정의 시 변수
- Argument = 호출 시 실제 값

### 2. `self`의 역할

```python
class BankAccount:
    def __init__(self, account_number, initial_balance=0):
        # self = 객체 접근을 위한 다리(참조)
        self.account_number = account_number  # 다리를 통해 속성 설정
        self.__balance = initial_balance      # 다리를 통해 속성 설정
    
    def deposit(self, amount):
        # self 다리를 통해 속성 접근
        self.__balance += amount
```

**핵심 이해:**
- `self` = 현재 객체 인스턴스 참조
- Java의 `this`와 유사하지만 명시적으로 선언
- 첫 번째 매개변수로 항상 선언
- 호출 시 자동으로 전달됨

### 3. 접근 제어자

```python
self.account_number      # Public (언더스코어 0개)
self._last_transaction   # Protected (언더스코어 1개)
self.__balance          # Private (언더스코어 2개)
```

**핵심 이해:**
- `_` 갯수로 접근 제어 표현
- `self`는 접근 제어자가 아님
- `self`는 객체 참조, `_`는 접근 제어

### 4. Java vs Python 비교

| 항목 | Java | Python |
|------|------|--------|
| **객체 참조** | `this` (키워드, 자동) | `self` (매개변수, 명시적) |
| **접근 제어** | `public`, `private`, `protected` | `_` 갯수로 표현 |
| **메서드 선언** | `public void method()` | `def method(self, ...)` |

### 5. 인자 순서 규칙

```python
def 메서드명(self, 실제인자1, 실제인자2, ...):
    # 1번째: self (항상, 자동 전달)
    # 2번째부터: 실제 전달하는 값들
    pass
```

**핵심 규칙:**
- 첫 번째 인자 = `self` (항상)
- 두 번째 인자부터 = 실제 값들
- `self` 생략 시 `TypeError` 발생
