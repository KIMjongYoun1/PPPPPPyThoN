# 자주 나오는 에러 & 해결

---

## 1. `command not found: pip`

**원인**: pip가 PATH에 없음 (가상환경 미활성화 또는 pip 미설치)

**해결**:
```bash
# 방법 1: python -m pip 사용
python -m pip install pandas numpy scipy scikit-learn statsmodels

# 방법 2: 가상환경 활성화 후 pip
source my_python_env/bin/activate   # macOS/Linux
pip install pandas numpy ...
```

---

## 2. `source: no such file or directory: .../activate`

**원인**: 경로 오타 (흔히 `User` vs `Users`)

**잘못된 예**: `/User/ryankim/...`  
**올바른 예**: `/Users/ryankim/...`

**해결**:
```bash
cd /Users/ryankim/Python
source my_python_env/bin/activate
```

---

## 3. `ModuleNotFoundError: No module named 'pandas'`

**원인**: pandas 등 패키지 미설치

**해결**:
```bash
source my_python_env/bin/activate
python -m pip install pandas numpy scipy scikit-learn statsmodels
```

---

## 4. `ModuleNotFoundError: No module named 'statsmodels'`

**원인**: 24, 25번(로지스틱·다중선형회귀)에 statsmodels 필요

**해결**:
```bash
python -m pip install statsmodels
```

---

## 5. `python3: command not found` / `python: command not found`

**원인**: Python 미설치 또는 PATH 미설정

**해결**:
1. [Python 공식](https://www.python.org/downloads/)에서 설치
2. `01_가이드문서/01_python_setup.md` 참고

---

## 6. `FileNotFoundError` / `No such file or directory`

**원인**: 잘못된 경로에서 실행, 또는 파일 경로 오타

**해결**:
```bash
# 반드시 프로젝트 루트에서 실행
cd /Users/ryankim/Python
python 03_기초부터_심화/01_변수와출력.py
```

---

## 7. `Operation not permitted` (pip 설치 시)

**원인**: 권한·샌드박스 제한

**해결**:
- 가상환경 사용 시 발생하지 않아야 함
- `python -m pip install --user ...` 시도
- 또는 터미널을 새로 열고 다시 시도

---

## 8. 터미널을 여는 방법

| OS | 방법 |
|----|------|
| **Mac** | Spotlight(⌘+Space) → "터미널" 입력 |
| **Windows** | 시작 메뉴 → "명령 프롬프트" 또는 "PowerShell" |
| **VS Code / Cursor** | 메뉴 Terminal → New Terminal |
