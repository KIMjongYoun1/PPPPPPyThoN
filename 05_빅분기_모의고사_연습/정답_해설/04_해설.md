# 04. HRdata 이직 예측 — 해설

## 데이터 구조 (4개 CSV)

경로: `BASE/HRdata/` — **train/test가 파일 단위로 쪼개진 형태** (열 = 세로, 행 = 가로)

| 파일 | 역할 |
|------|------|
| **X_train.csv** | 학습용 **입력** — 특성 열(세로)들 + `enrollee_id` |
| **y_train.csv** | 학습용 **정답** — 보통 열(세로)이 적음 (`target` 등) |
| **X_test.csv** | 시험용 **입력** — 같은 형태의 특성 + `enrollee_id` |
| **y_test.csv** | 연습용 **정답** — 채점기에 안 줄 수도 있음; 로컬에서 accuracy 확인용 |

- **행(가로 줄)** 은 지원자 한 명 단위로 train / test가 서로 다름 (다른 사람 표본).
- **y_train** 은 `y_train["target"]` 처럼 **Series(1차원)** 로 두고 `model.fit(X_train, y_train)` 에 넣는다.

---

## 풀이 흐름

1. **Step 3**: 네 파일 `read_csv` 로 로드.
2. **Step 4**: `enrollee_id` 는 제출·식별용 → `X_train`, `X_test` 에서 **drop**. `y_train = y_train["target"]`.
3. **Step 5**: `object` 열(세로) → `LabelEncoder` 로 숫자화. 결측은 `fillna(0)` 등.
4. **Step 6**: **분류** → `RandomForestClassifier`, `fit(X_train, y_train)`.
5. **Step 7**: `predict(X_test)` → `submission.csv` 열: `enrollee_id`, `target`, `index=False`.

---

## 핵심 포인트

| 주제 | 내용 |
|------|------|
| **왜 Classifier?** | `target` 이 **0/1 (이직 여부)** → **갈래 맞추기**이므로 `RandomForestClassifier`. (`Regressor`는 연속 숫자 예측용) |
| **범주형 자동 선택** | `X_train.select_dtypes(include=["object"]).columns` — 글자 열(세로)을 한꺼번에 골라 인코딩. 열 이름을 일일이 써도 됨. |
| **`fit` 의미** | train(또는 아래 concat) 데이터를 보고 **글자→숫자 번호표** 또는 **모델 가중치** 등 **규칙을 정함**. |
| **`pd.concat(..., axis=0)`** | **행(가로 줄) 방향으로 아래에 이어 붙임** — 같은 열(세로)의 train 칸 + test 칸을 **한 긴 세로 시리즈**로 만들 때 사용. |
| **연습용 정확도** | `y_test` 가 있으면 `(pred == y_test["target"].values).mean()` 등으로 확인. |
| **AUC 제출** | 문제에서 확률 제출이면 `predict_proba(X_test)[:, 1]` |

## LabelEncoder: train만 fit 할 때 vs train+test 합쳐 fit

| 방식 | 동작 | 언제 깨지나 |
|------|------|-------------|
| train만 `fit_transform` / test `transform` | 원칙적으로는 **train에 나온 범주만** 번호표에 있음 | **test에만 있는 글자**(예: `city_171`)가 있으면 `ValueError: previously unseen labels` |
| `concat([X_train[col], X_test[col]], axis=0)` 후 `fit` | train·test **그 열에 등장하는 모든 글자**가 번호표에 들어감 | 인코딩 에러는 피하기 쉬움 |

- **미리 test에만 있는 범주가 있는지**는 데이터를 안 열어보면 모를 수 있음 → 실행해 보고 에러 나면 이 패턴으로 바꾸거나, 처음부터 범주형에 적용해도 됨.
- 엄밀한 실무(누수 논의)와 달리, **시험 제출·이 연습 데이터**에서는 자주 쓰는 **실용적 타협**.

**대안**: `OneHotEncoder(handle_unknown="ignore")`, 미지의 값을 특정 코드로 매핑 등.

## 주의사항 (어디에 적용되는지)

| 말 | 적용하는 곳 |
|----|-------------|
| **`enrollee_id` 제거** | **Step 4** — 모델 입력 `X`에는 넣지 않고, **Step 7** `submission["enrollee_id"]` 에만 사용. |
| **범주형 인코딩** | **Step 5** — `object` 열. 숫자인데 범주인 열은 `dtypes` / `head` 로 따로 처리. |
| **train+test concat fit** | **Step 5** `LabelEncoder` 루프 안 — `previously unseen labels` 방지. |
| **분류 모델** | **Step 6** — `RandomForestClassifier` (회귀 아님). |
| **제출 파일** | **Step 7** — `enrollee_id`, `target`, `to_csv(..., index=False)`. |

---

## 자주 하는 실수

- `from sklearn.preprocessing import LabelEncoder` — `ensemble` 이 아님.
- `submission` 열 이름 오타 (`target` 아닌 다른 이름).
- `y_train` 을 DataFrame 그대로 두면 `fit` 에서 shape 경고/에러 날 수 있음 → `y_train["target"]` 으로 Series.
