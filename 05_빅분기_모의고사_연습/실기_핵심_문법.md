## 2. 1유형 — 집계·전처리

### 요구사항 → 사용 함수 → 답 (매핑표)

| 요구사항 | 사용할 것 | 결과 |
|----------|-----------|------|
| **A별 B 합계** (예: 나라별 골득점) | `df.groupby("A")["B"].sum()` | Series |
| **A별 B 평균** | `df.groupby("A")["B"].mean()` | Series |
| **A별 행 개수** | `df.groupby("A").size()` | Series |
| **상위 N개** (예: 상위 10개 채널) | `value_counts().head(10)` 또는 `sum().nlargest(10)` | Series |
| **가장 많은 값** (최빈값) | `value_counts().index[0]` | 값 1개 |
| **가장 큰/작은 행** (예: ratio 최대인 영상) | `sort_values("컬럼", ascending=False).iloc[0]` | 행 1개 |
| **조건 만족하는 개수** | `len(df[조건])` | 숫자 |
| **이름에 'X' 포함** (대소문자 무시) | `df["컬럼"].str.lower().str.contains("x")` | boolean |
| **요일별 개수** | `df["날짜"].dt.day_name()` + `groupby` | DataFrame |
| **A별 B별 개수 → 표** | `groupby(["A","B"]).size()` + `pivot` | DataFrame |
| **비율/ratio 구해서 정렬** | `df["ratio"] = A/B` → `sort_values("ratio")` | DataFrame |
| **채널명 바꾼 케이스** | `drop_duplicates(["A","B"]).B.value_counts()` → `>1` 개수 | 숫자 |
| **Years split 후 4자리 체크** | `str.split("-")` → `apply(체크함수)` | boolean |
| **N회 이상인 개수** | `(df[["A","B"]].value_counts() >= N).sum()` | 숫자 |
| **view_count=0 제외** | `df[df["view_count"] != 0]` (ratio 구할 때) | DataFrame |

## 3. 2유형 — 맞추기 문제 (A/B·숫자 예측)

**용어만 먼저:** 책·코드에서 **y** = **맞출 정답**(한 열), **X** = **정답 빼고 모델에 넣을 정보**(표). **분류** = A/B·예/아니오처럼 **갈래** 맞추기, **회귀** = **숫자**(금액·점수 등) 맞추기.

### 판단 체크리스트 (문제 읽기 → 무엇을 넣을지 정하기)

문제 글과 **표를 직접 본 다음** 아래 순서로 정하면 헷갈림이 줄어듦.

#### ① 문제지에서 먼저 정할 것

| 이렇게 나오면 | 이렇게 이해 |
|----------------|-------------|
| “○○을 예측”, “맞출 값은 …” | 그 열 이름 = **정답(y)** → `y_train = train["그열이름"]` |
| 제출 파일 예시(열 이름 순서) | 저장할 때도 **그 이름·순서** 맞출 것 |
| “AUC”, “분류”, 맞다/틀리다 쪽 | **갈래 맞추기** → `RandomForestClassifier`, 필요하면 `predict_proba` |
| “RMSE”, “오차”, 연속된 숫자 | **숫자 맞추기** → `RandomForestRegressor` |
| 문제에 숫자가 적혀 있음 (`n_estimators` 등) | **적힌 대로** 쓰기 |

#### ② 파일 불러온 직후 (가능하면 매번)

```python
train.head()
test.head()
train.columns.tolist()
test.columns.tolist()
train.dtypes
# train.info()  # 시간 되면
```

| 표를 보면 | 뜻 |
|-----------|-----|
| **학습용(train)** 에만 있고 **시험용(test)** 에 없는 열 | 보통 **정답** → 학습용 입력 표에서는 **그 열 빼기**(`drop`) |
| `ID`, `enrollee_id` 같은 **번호 열** | **제출할 때 줄 번호**로 쓰고, 맞추는 데 쓸 정보에서는 **빼기** |
| test에 **번호 열이 없음** | `test_ids = test.index` 처럼 **위에서부터 0,1,2…** 를 ID로 (문제·샘플 제출 양식 확인) |

#### ③ “글자 칸” vs “숫자 칸” (대략 이렇게 보면 됨)

| 글자 칸처럼 보이면 (나중에 숫자로 바꿔야 할 후보) | 숫자 칸으로 두는 경우 |
|------------------------------------------|----------------------|
| pandas가 **object** 라고 찍는 열 (글자) | 나이, 금액처럼 **크고 작음**이 의미 있는 숫자 |
| 남/여, 예/아니오, 지역 이름처럼 **이름만 다른 값** | BMI, 사용 기간, 잔액 등 |
| 숫자인데 종류가 몇 개뿐 (1=서울, 2=부산…) | 잘 모르겠으면 `head()` / `unique()` 로 실제 값 확인 |

**글자 열만 한꺼번에 고르기 (빠를 때):**

```python
cat_cols = X_train.select_dtypes(include=["object"]).columns.tolist()
```

직접 리스트를 쓸 때는 표를 보고 `["sex", "region", ...]` 처럼 정하면 됨.

#### ④ 실수 안 나게 한 줄씩

- **정답(y)은 “맞추는 재료(X)”에 넣지 않기** — 넣으면 정답을 미리 보는 것과 같음.
- **글자→숫자(LabelEncoder)**: 보통 학습용만 `fit_transform`, 시험용은 `transform`. 다만 **시험용에만 있는 범주**가 있으면 에러 → 그 열은 **train+test 합쳐 `fit`** (HRdata `city` 등).
- **빈 칸**: 문제에 없으면 `fillna(0)` 같이 단순히 채워도 통과하는 경우 많음.
- **시간 없을 때**: 세부 조절보다 **제출 파일 형식 맞춰서 저장까지**가 먼저.

---

### 요구사항 → 쓸 코드 → 나오는 것 (매핑표)

| 하려는 일 | 쓸 것 | 나오는 것 |
|-----------|--------|-----------|
| **train/test에서 정답·입력 나누기** | `drop(columns=["정답열","ID"])` 등 | X_train, y_train |
| **X_train / X_test 가 이미 파일로 나뉨** | ID만 빼기 `drop(columns=["ID"])` | X_train, X_test |
| **글자 칸을 숫자로** | `LabelEncoder().fit_transform` | 숫자로 바뀐 표 |
| **빈 칸 채우기** | `fillna(0)` | 빈 곳이 0 등으로 채워진 표 |
| **갈래 맞추기 예측** | `RandomForestClassifier().fit().predict()` | 0/1 등 |
| **숫자 맞추기 예측** | `RandomForestRegressor().fit().predict()` | 연속된 숫자 |
| **확률로 제출 (AUC 등)** | `predict_proba(X)[:, 1]` | 0~1 사이 |
| **제출 파일 만들기** | `pd.DataFrame({"ID": ids, "열이름": pred}).to_csv(..., index=False)` | submission.csv |
| **답이 여러 글자일 때 (약 이름 등)** | 정답도 LabelEncoder → 끝에 `inverse_transform` | 다시 글자로 |

### 정답(y)과 입력(X) 나누기

```python
# train.csv / test.csv 가 따로 있을 때
y_train = train["맞출열이름"]
X_train = train.drop(columns=["맞출열이름", "ID"])
X_test = test.drop(columns=["ID"])
test_ids = test["ID"]

# X_train.csv / X_test.csv 처럼 이미 나뉜 경우 (HRdata, diabetes 등)
test_ids = X_test["ID"]  # 또는 X_test["enrollee_id"] 등
X_train = X_train.drop(columns=["ID"], errors="ignore")
X_test = X_test.drop(columns=["ID"])
y_train = y_train["target"]  # 파일마다 열 이름 확인
```

### 글자로 된 열만 자동으로 고르기

```python
cat_cols = X_train.select_dtypes(include=["object"]).columns.tolist()
```

### LabelEncoder (글자 → 숫자로 바꾸기)

```python
from sklearn.preprocessing import LabelEncoder

for col in ["Geography", "Gender"]:   # 글자 칸 이름
    le = LabelEncoder()
    X_train[col] = le.fit_transform(X_train[col].astype(str))
    X_test[col] = le.transform(X_test[col].astype(str))
```

**test에만 나오는 글자 값**이 있으면 `transform`에서 에러(`previously unseen labels`) → 그 열은 **train+test를 이어 붙여서 `fit`**:

```python
for col in cat_cols:
    le = LabelEncoder()
    both = pd.concat([X_train[col], X_test[col]], axis=0).astype(str)
    le.fit(both)
    X_train[col] = le.transform(X_train[col].astype(str))
    X_test[col] = le.transform(X_test[col].astype(str))
```

### 비어 있는 칸(결측) 채우기

```python
X_train = X_train.fillna(0)
X_test = X_test.fillna(0)
```

### 갈래 맞추기 모델 (분류)

```python
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X_train, y_train)
pred = model.predict(X_test)
```

### 숫자 맞추기 모델 (회귀)

```python
from sklearn.ensemble import RandomForestRegressor

model = RandomForestRegressor(n_estimators=100, random_state=42)
model.fit(X_train, y_train)
pred = model.predict(X_test)
```

### 제출 파일 저장

```python
submission = pd.DataFrame({"ID": test_ids, "맞출열이름": pred})
submission.to_csv("submission.csv", index=False)
```

### 정답이 여러 종류일 때 (y도 글자→숫자→다시 글자)

```python
le_y = LabelEncoder()
y_enc = le_y.fit_transform(y_train)
model.fit(X_train, y_enc)
pred_enc = model.predict(X_test)
pred = le_y.inverse_transform(pred_enc)
```

## 4. 3유형 — 통계

### 요구사항 → 사용 함수 → 답 (매핑표)

| 요구사항 | 사용할 것 | 결과 |
|----------|-----------|------|
| **F검정 통계량** (분산 비) | `np.var(..., ddof=1)` → 큰분산/작은분산 | round(값, 3) |
| **합동 분산** | `((n1-1)*v1+(n2-1)*v2)/(n1+n2-2)` | round(값, 3) |
| **t검정 p-value** | `stats.ttest_ind(g1, g2)` | round(p_value, 3) |
| **오즈비** | `logit().fit()` → `np.exp(params["x"])` | round(값, 3) |
| **유의한 변수 회귀계수** | `pvalues < 0.05` → `params[sig.index]` | round(값, 3) |
| **R² (결정계수)** | `ols().fit()` → `model.rsquared` | round(값, 3) |
| **유의한 변수 회귀계수 합** | `params[pvalues<0.05].sum()` (Intercept 제외) | round(합, 3) |
| **로그 변환** (23번) | `np.log(값)` — F/t검정 전 | 변환된 시리즈 |

---

### F검정, t검정 (scipy)

```python
import numpy as np
from scipy import stats

# 1) 두 집단 분리, 로그 변환 (23번)
cls_1 = df[df["Classification"] == 1]["Resistin"]
cls_2 = df[df["Classification"] == 2]["Resistin"]
g1 = np.log(cls_1)
g2 = np.log(cls_2)

# 2) 분산 (ddof=1: 표본)
var1 = np.var(g1, ddof=1)
var2 = np.var(g2, ddof=1)
f_stat = max(var1, var2) / min(var1, var2)  # 큰/작은

# 3) 합동 분산
n1, n2 = len(g1), len(g2)
var_pooled = ((n1-1)*var1 + (n2-1)*var2) / (n1+n2-2)

# 4) t검정
t_stat, p_value = stats.ttest_ind(g1, g2)
```

### 로지스틱 회귀 (statsmodels)

```python
from statsmodels.formula.api import logit

# 전제: df에 target, x1, x2 있고 결측 제거 (logit은 NaN 불가)
df = df.dropna()
model = logit("target ~ x1 + x2", data=df).fit()
odds_ratio = np.exp(model.params["x1"])
sig = model.pvalues[model.pvalues < 0.05].drop("Intercept", errors="ignore")
```

### 다중선형회귀 (statsmodels)

```python
from statsmodels.formula.api import ols

# 전제: df에 y, x1, x2, x3 있고 결측 제거. 컬럼명에 공백 있으면 rename
model = ols("y ~ x1 + x2 + x3", data=df).fit()
r2 = model.rsquared
sig_coefs = model.params[model.pvalues < 0.05].drop("Intercept", errors="ignore")
coef_sum = sig_coefs.sum()
```

## 6. 자주 틀리는 것

| 흔한 실수 | 이렇게 하기 |
|-----------|-------------|
| 글자→숫자 할 때 시험용(test)까지 `fit` | 학습용만 규칙 익히고, 시험용은 `transform`만 |
| 열 이름 대소문자 | `CustomerId` vs `customerId`, `ID` vs `id` 문제지·표와 똑같이 |
| 저장할 때 줄 번호가 한 줄 더 붙음 | `to_csv(..., index=False)` |
| 3유형을 sklearn으로 풀려 함 | 통계 문제는 `statsmodels` (logit, ols) |
| `round(값, 3)` | 소수 **셋째 자리**까지 |
| 나눗셈(비율) 할 때 | 조회수·좋아요가 0인 행은 빼기 (0으로 나누기 방지) |
| logit/ols | 빈 칸 있으면 먼저 `df.dropna()` |
| 피벗(pivot) 할 때 | 표에 실제로 있는 열 이름 쓰기 (예: trending_date2) |
